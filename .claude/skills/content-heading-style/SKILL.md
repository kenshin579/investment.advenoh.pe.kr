# Content Heading Style

`contents/**/index.md` 파일의 Heading 번호 체계를 일관되게 유지하는 Skill이다.

## Heading 번호 규칙

모든 Heading에는 계층적 번호를 부여한다:

- **H1**: `# N. 제목` (예: `# 1. 들어가며`)
- **H2**: `## N.M 제목` (예: `## 1.1 Topic 구조`)
- **H3**: `### N.M.K 제목` (예: `### 1.1.1 계층적 네이밍`)

번호는 각 레벨별로 1부터 시작하며, 상위 Heading이 바뀌면 하위 번호는 1로 초기화된다.

## 동작 방식

`contents/` 디렉토리의 index.md 파일을 작성하거나 수정할 때 다음을 수행한다:

### 1. 새 글 작성 시

Heading을 작성할 때 반드시 아래 번호 체계를 적용한다:

```markdown
# 1. 들어가며
# 2. 개발 환경
# 3. 주제 제목
## 3.1 소제목 A
### 3.1.1 세부 항목
### 3.1.2 세부 항목
## 3.2 소제목 B
# 4. 마무리
# 5. 참고
```

### 2. 기존 글 수정 시

Heading을 추가/삭제/이동한 후 전체 번호를 재정렬한다.

**변환 대상 패턴:**

| 잘못된 패턴 | 올바른 패턴 |
|------------|-----------|
| `# 제목` (번호 없음) | `# N. 제목` |
| `# X장. 제목` | `# N. 제목` |
| `## 제목` (번호 없음) | `## N.M 제목` |
| `### 제목` (번호 없음) | `### N.M.K 제목` |
| `# 1.제목` (공백 없음) | `# 1. 제목` |

### 3. 검증

수정 완료 후 아래 조건을 확인한다:

- 모든 H1이 `# 숫자. 제목` 형식인지
- 모든 H2가 `## 숫자.숫자 제목` 형식인지
- 모든 H3이 `### 숫자.숫자.숫자 제목` 형식인지
- 번호가 순차적으로 증가하는지
- 상위 Heading이 바뀔 때 하위 번호가 1로 초기화되는지

## 예외 사항

다음은 변환 대상이 **아니다**:

- YAML frontmatter (`---` 블록) 내부
- 코드 블록(```)  내부의 `#`
- 인라인 코드 내부의 `#`
