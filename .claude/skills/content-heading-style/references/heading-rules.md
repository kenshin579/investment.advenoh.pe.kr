# Heading 번호 체계 상세 규칙

## 번호 형식

### H1 (대주제)
- 형식: `# N. 제목`
- N은 1부터 시작, 순차 증가
- `#`과 번호 사이 공백 1개, 번호와 `.` 붙임, `.`과 제목 사이 공백 1개
- 예: `# 1. 들어가며`, `# 2. 개발 환경`

### H2 (중주제)
- 형식: `## N.M 제목`
- N은 상위 H1의 번호, M은 1부터 시작
- `##`과 번호 사이 공백 1개, `N.M`과 제목 사이 공백 1개
- 예: `## 3.1 소제목`, `## 3.2 소제목`

### H3 (소주제)
- 형식: `### N.M.K 제목`
- N.M은 상위 H2의 번호, K는 1부터 시작
- `###`과 번호 사이 공백 1개, `N.M.K`과 제목 사이 공백 1개
- 예: `### 3.1.1 세부 항목`, `### 3.1.2 세부 항목`

## 번호 초기화 규칙

- 새로운 H1이 나오면 → H2 번호는 1로 초기화
- 새로운 H2가 나오면 → H3 번호는 1로 초기화

예시:
```markdown
# 1. 첫 번째 대주제
## 1.1 소주제
### 1.1.1 세부
### 1.1.2 세부
## 1.2 소주제
### 1.2.1 세부        ← H3 번호 1로 초기화
# 2. 두 번째 대주제
## 2.1 소주제          ← H2 번호 1로 초기화
### 2.1.1 세부        ← H3 번호 1로 초기화
```

## 일반적인 블로그 글 구조

```markdown
---
title: "제목"
date: "2024-01-15"
tags: ["tag1", "tag2"]
---

# 1. 들어가며
# 2. 개발 환경
# 3. 본문 주제
## 3.1 소주제 A
### 3.1.1 세부 항목
### 3.1.2 세부 항목
## 3.2 소주제 B
# 4. 마무리
# 5. 참고
```

## 잘못된 패턴과 수정 방법

### 번호 없는 Heading
```markdown
# 들어가며          → # 1. 들어가며
## MQTT란           → ## 1.1 MQTT란
### 한 줄 정의       → ### 1.1.1 한 줄 정의
```

### "X장." 형식
```markdown
# 1장. MQTT 개요    → # 1. MQTT 개요
# 0장. 시작하기     → # 1. 시작하기
```

### 공백 누락
```markdown
# 1.들어가며        → # 1. 들어가며
## 1.1소제목        → ## 1.1 소제목
```

### 번호 불연속
```markdown
# 1. 들어가며
# 3. 본문           → # 2. 본문 (순차적으로 수정)
```

## 예외: 변환하지 않는 영역

1. **YAML frontmatter**: `---`로 둘러싸인 메타데이터 블록
2. **코드 블록**: 백틱 3개로 둘러싸인 영역 내부의 `#`
3. **인라인 코드**: 백틱으로 둘러싸인 텍스트 내부의 `#`
4. **HTML 주석**: `<!-- -->` 내부
